## 目标

修复 `1920x1080` 真实 Godot .NET 窗口中斗法手牌操作按钮越界的问题，并用 Godot MCP 真实点击验证斗法能继续推进到奖励、构筑和返回灵脉。

## 根因

本次不是单个按钮坐标错误，而是多个 Control 最小尺寸共同把斗法页撑高：

1. `AddCombatHeader` 中的标题 Label 继承默认自动换行，在 HBox 最小宽度下把 `斗法` 两个字按竖向高度计算，头部被撑到约 `168` 高。
2. 资源徽章和按钮没有明确纵向收缩，容器会参与额外高度分配。
3. 按钮纹理九宫格边距写死为 `32`，实际按钮高度变为 `87`，但布局预算仍按 `button_height=60` 计算。
4. 面板纹理九宫格边距写死为 `48`，推高面板最小高度。
5. 斗法记录文本无限增长，会把右侧记录面板和上半区继续撑高，导致多次出牌后按钮再次越界。

## 修复内容

| 文件 | 内容 |
| --- | --- |
| `scripts/Main.cs` | 斗法标题关闭自动换行；标题行、资源徽章、按钮设置纵向收缩；斗法记录只显示最近配置行数；面板和按钮纹理边距改为读取配置。 |
| `scripts/GameSpecLoader.cs` | 补充 `button_texture_margin`、`panel_texture_margin`、`combat_log_visible_lines` 默认值。 |
| `data/game_spec/ascension_forge.prototype.json` | 增加 1080p UI 配置：按钮纹理边距、面板纹理边距、斗法记录可见行数。 |

## Godot MCP 实测

真实窗口：`1920x1080`

流程：

1. 主菜单点击 `新开灵契`。

4. 多次点击 `施展`，并点击 `收势回合`。
5. 打赢斗法后进入 `斗法奖励`。
6. 点击 `收取` 进入 `构筑洞府`。
7. 点击 `返回灵脉` 回到灵脉图。
