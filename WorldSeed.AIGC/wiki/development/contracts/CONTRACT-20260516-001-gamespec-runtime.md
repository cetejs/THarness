## 目标

定义 GameSpec 数据层到 Godot 运行时的交接契约。后续开发必须通过该契约读取角色、卡牌、地图、战斗、奖励、构筑、素材和存档相关数据。

## 输入

GameSpec 至少需要提供以下数据集合：

| 数据集合 | 内容 |
| --- | --- |
| meta | 游戏 ID、版本、内容阶段。 |
| ui_flow | 界面 ID、按钮动作、跳转关系。 |
| assets | UI、卡框、图标、地图节点、角色、敌人的素材引用。 |
| tags | 卡牌、辅助、装备、状态和天赋共用标签。 |
| status_effects | 战斗状态定义。 |
| characters | 角色、初始牌组、初始装备、起始天赋。 |
| cards | 攻击、防御、技能、辅助、诅咒。 |
| equipment | 装备槽位、基础属性、词缀引用。 |
| affixes | 装备、地图、奖励和敌人的词缀规则。 |
| talents | 天赋节点、前置关系、效果。 |
| enemies | 敌人属性、意图、状态抗性。 |
| encounters | 敌人组合、难度、奖励池。 |
| events | 事件选项、代价、收益。 |
| shops | 商品池、锻造服务和价格规则。 |
| map_generation | 章节地图层数、节点类型、连线规则。 |
| chapters | 章节主题、敌人池、事件池、Boss。 |
| rewards | 奖励池、金币范围、卡牌池、装备池。 |
| save_fields | 一局游戏需要保存的状态字段。 |

## 输出

Godot 运行时读取 GameSpec 后，应能生成或驱动：

- 主菜单、角色选择、地图、战斗、奖励、构筑管理、事件、商店、章节结算和本局结算的 UI 数据。
- 玩家初始局内状态。
- 地图节点和可达路径。
- 战斗遭遇。
- 手牌、抽牌堆、弃牌堆和消耗区。
- 卡牌效果、状态效果、敌人意图和回合结果。
- 奖励候选。
- 辅助卡插槽兼容判断。
- 装备和天赋效果。
- 存档和继续游戏所需状态。

## 错误处理

开发期允许直接失败并输出明确错误，不需要掩盖错误：

| 错误 | 处理方式 |
| --- | --- |
| ID 引用不存在 | 阻止加载，报告缺失 ID 和引用来源。 |
| 标签不存在 | 阻止加载，报告卡牌、辅助、装备或天赋 ID。 |
| 素材引用不存在 | 开发期报告错误；若配置了占位素材，可回退到占位素材。 |
| UI 跳转目标不存在 | 阻止加载，报告来源界面和动作。 |
| 奖励池为空 | 阻止进入对应奖励流程。 |
| 起始牌组为空 | 阻止开始新局。 |
| Boss 遭遇缺失 | 阻止生成章节地图。 |

## 验证方式

后续实现前必须具备以下验证：

1. GameSpec 根模块完整性检查。
2. 所有 ID 引用完整性检查。
3. 标签兼容性检查。
4. UI 跳转完整性检查。
5. 素材引用存在性检查。
6. 第一阶段一局链路检查：角色选择到 Boss 结算没有断链。

## 禁止事项

- 禁止 Godot 运行时硬编码卡牌、敌人、奖励、地图或素材路径。
- 禁止在 UI 层直接写死战斗数值。
- 禁止辅助卡绕过标签兼容规则。
- 禁止装备、天赋、词缀直接修改运行时全局状态而不经过效果系统。
- 禁止在未记录契约变更的情况下修改 GameSpec 字段含义。
