## 目标

定义 v0.3 设计稿到 Godot .NET/C# 运行时 UI 的交接契约。后续 UI 修正必须先满足本契约，不能只凭“能进入界面”或“无编译错误”判定完成。

## 输出

每个玩家可见界面必须输出或具备：

- 界面 ID 和显示名。
- 目标设计稿路径。
- 目标运行分辨率。
- 区域布局比例。
- 主要按钮、状态和跳转。
- 数据来源。
- 运行时截图或截图验收要求。
- 差异清单和允许偏差。

## 禁止事项

- 禁止用全局滚动框掩盖布局问题。
- 禁止把 1280x720 设计稿直接等比放大后视为完成。
- 禁止只修单个控件而不回到目标设计稿做整体截图对比。
- 禁止在 UI 脚本中硬编码卡牌、敌人、奖励、法器、天赋或 Boss 展示。
- 禁止把装饰背景放到会遮挡玩家操作区的层级。

## 验证方式

- 运行时窗口为 `1920x1080` 或按同一逻辑画布等比缩放。
- 每个关键界面都有目标图和实机截图对比。
- P0 界面的差异记录必须为空或被明确接受。
- 构建、数据、smoke 和文本编码验证通过。
