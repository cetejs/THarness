# 策划触发索引

| 用户表达 | 触发方法卡 | AI 动作 |
| --- | --- | --- |
| 我想做一个很爽的玩法 | `art-of-game-design`, `game-feel` | 追问爽感来源、首次爽点、输入反馈和风险代价。 |
| 我想让玩家可以修仙 | `designing-games`, `game-mechanics` | 转译身份幻想、资源、功法、成长、突破和局内循环。 |
| 我希望有宗门玩法 | `art-of-game-design`, `designing-games`, `game-mechanics` | 追问宗门在局内如何选择、如何改变战斗、是否有代价。 |
| 我想加一个系统 | `designing-games`, `game-mechanics` | 追问系统服务的体验、核心循环、资源流和决策点。 |
| 战斗要更有手感 | `game-feel` | 追问输入响应、命中反馈、镜头、音效、节奏和可读性。 |
| 玩家可能看不懂 | `gamers-brain` | 检查引导时机、UI 层级、术语负担和首次学习路径。 |
| 这个想法值不值得做 | `sprint`, `mom-test` | 设计最小原型、观察行为指标和非诱导式验证问题。 |

## 使用规则

- 触发索引只决定优先读取哪张卡，不替用户做设计决策。
- 同时命中多张卡时，按 `phase-index.md` 的阶段顺序输出。
- 如果用户表达存在多种解释，先列出解释差异，再提问。
- 方法卡状态为 `pending-source` 时，只能按卡片结构组织讨论，不得声称该规则已由书籍原文验证。
