# The Gamer's Brain 方法卡

## 覆盖模块

用户理解。

## 来源状态

partial。

## 适用阶段

降低理解成本。

## 基础数据源

- 作者官网：`https://thegamersbrain.com/`

## 卡片功能

用游戏 UX 和认知心理学检查玩家是否能看懂、记住、判断、学习并持续投入。

## 提炼目标

检查玩家是否能理解目标、规则、反馈和下一步行动，降低 UI、术语和引导负担。

## 原则卡

- 认知限制优先：设计必须尊重玩家的注意、记忆、感知和学习限制。
- 全流程 UX：玩家体验从听说游戏开始，覆盖菜单、引导、局内反馈和长期进度。
- 可用性和吸引力并重：既要容易理解，也要能持续产生投入。
- 用研究闭环校准：UX 问题应通过观察、用户研究、数据和迭代验证。

## 提问卡

- 玩家第一次遇到机制时看见什么？
- 玩家如何知道自己可以做这个选择？
- UI 是否直接表达收益、风险和当前状态？
- 哪些术语会让新玩家误解？
- 机制是否能在三分钟内被理解？

## 转译卡

- 复杂规则 -> 可见状态、渐进引导和局内示例。
- 多种选择 -> 对比维度、代价提示和推荐默认。
- 长期成长 -> 当前收益、下一目标和变化预告。

## 检查卡

- 是否能在 3 分钟内被理解？
- 玩家是否知道下一步该做什么？
- 玩家是否能从反馈中修正行为？
- UI 是否服务决策，而不是只展示设定名词？
- 是否降低了注意、记忆、术语和界面层级负担？
- 是否同时检查了 usability 和 engage-ability？
