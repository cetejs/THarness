# 提问规则

出现以下情况必须提问：

- 目标不明确。
- 范围边界不明确。
- 存在多个合理解释。
- 需要选择目标项目路径。
- 需要判断内容属于通用知识还是项目事实。

提问前必须先说明为什么该问题会影响后续结果。

## 游戏策划追问

出现以下情况时，必须读取 `game-design-methodology.md` 和 `../method-cards/trigger-index.md`：

- 用户只说了世界观、题材、设定或系统名词，但没有说明玩家局内做什么。
- 用户使用“爽”“有深度”“修仙感”“策略性”“战斗手感”等模糊体验词。
- 用户提出功能点，但没有说明玩家为什么会反复使用。
- 用户提出玩法方向，但没有说明理解成本、反馈方式或验证原型。

游戏策划追问优先顺序：

1. 玩家最想扮演什么？
2. 玩家最常重复的动作是什么？
3. 玩家什么时候第一次感到爽或理解玩法成立？
4. 爽感来自爆发、连击、成长、压制、反杀还是选择差异？
5. 这个机制的风险、代价或机会成本是什么？
6. 玩家如何通过 UI、特效、音效、镜头或关卡理解它？
7. 最小原型要验证什么真实行为？
