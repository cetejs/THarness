# 开发执行 Agent 索引

默认不读取所有 agent。按当前阶段选择一个或多个 agent。

| Agent | read_when |
| --- | --- |
| `coordinator` | 需要接收用户目标、确认开发是否可执行、同步阻塞或交付。 |
| `lead-developer` | 需要主程与用户沟通、拆分任务、分配角色、定义接口契约、集成验收或审核知识候选。 |
| `developer` | 需要实施明确的代码、文档、测试或配置改动；游戏角色默认继承该通用执行规则。 |
| `ui-developer` | 子任务主要涉及界面、HUD、菜单、提示、交互面板或 UI 状态流。 |
| `character-controller-developer` | 子任务主要涉及 3C：角色、镜头、输入、移动、手感或动画状态接入。 |
| `scene-developer` | 子任务主要涉及场景、关卡、触发器、出生点、导航或场景交互。 |
| `combat-developer` | 子任务主要涉及攻击、受击、技能、伤害、状态或战斗判定。 |
| `ai-developer` | 子任务主要涉及敌人行为、感知、巡逻、仇恨、目标选择或决策。 |
| `gameplay-systems-developer` | 子任务主要涉及背包、任务、存档、配置、事件流或全局玩法系统。 |
| `tools-developer` | 子任务主要涉及编辑器工具、批处理、导入导出、调试面板或内容管线。 |
| `technical-art-developer` | 子任务主要涉及特效、材质、Shader、动画事件或表现接入。 |
| `qa-verification` | 子任务主要涉及复现、回归、验收、失败路径或压力检查。 |
| `wiki-curator` | 需要判断开发结果是否沉淀到通用 wiki 或目标项目 wiki。 |

## 执行说明

主程阶段可以由当前会话承担。子任务执行阶段必须使用全新 SubAgent 或全新会话；运行环境无法启动真实 SubAgent 时，必须用新的任务上下文模拟隔离，并在交付物中说明限制。

游戏开发角色是通用角色定义。项目内具体路径、资源命名、模块边界和实现约定必须来自项目 wiki 子任务页或其他被允许读取的项目 wiki 页面。
