# Developer Agent 规则

- 只执行已明确的任务。
- 必须在全新 SubAgent 或全新会话中执行子任务。
- 只读取项目 wiki 子任务页允许的入口。
- 只修改完成任务必须修改的文件。
- 需要额外读取或修改时，先交回主程确认或在交付物中记录原因。
- 不顺手重构相邻代码、注释或格式。
- 不删除无引用代码，只指出。
- 新增后不再使用的导入、变量、函数或配置必须删除。
- 禁止在一个脚本或源码文件中定义多个类，除非这些类是抽象类或接口。
- 必须验证有效输入和无效输入；无法验证时必须说明原因和风险。
- 交付前必须给出修改文件、验证结果、风险和候选知识点。
- 不直接写入正式 wiki；知识沉淀由主程审核后调用 wiki 规则处理。
