# 交付规则

开发交付物的最小必填内容：

- 本轮目标。
- 项目 wiki 策划页面入口和命中章节。
- 项目 wiki 开发任务页入口。
- 实施内容。
- 修改文件。
- 验证结果。
- 自检触发判断和自检结果。
- 风险、限制或未完成事项。
- 子任务分配和采纳状态。
- Wiki 候选知识处理判断。

## 按需补充

命中以下情况时，必须补充对应内容：

| 情况 | 补充内容 |
| --- | --- |
| 涉及输入、参数或边界条件 | 无效输入测试结果。 |
| 发生方案取舍 | 关键决策。 |
| 实际比较并放弃过方案 | 被否决方案。 |
| 产生可复用认知 | 经验教训或候选知识点。 |
| 影响项目文档 | 建议文档更新。 |
| 影响 OneHarness 自身文档、规则、配置或工具 | 按 `self-check.md` 运行的自检命令和结果。 |
| 存在阻塞性歧义 | 需要用户确认的问题。 |
| 启用多个执行者或 SubAgent | 子任务分配、全新会话、文件归属、采纳记录和未采纳原因。 |
| 产生候选知识点 | 主程审核结果、写入或不写入理由、调用的 wiki 规则。 |

## 状态限制

- 验证未完成时，不能标记为完成。
- 有关键待确认问题时，不能把相关内容写成稳定事实。
- 临时调试过程不进入 wiki，只能保留在交付物或运行记录中。
- 子任务没有文档入口时，不能标记为已分配。
- 子任务不是全新 SubAgent 或全新会话执行时，必须说明运行环境限制和替代隔离方式。
- 候选知识点未经主程审核时，不能写入正式 wiki。
