# 游戏开发角色路由规则

游戏开发角色是通用 AIGC 固定能力。角色职责保持通用，项目内的具体模块事实、路径、命名、参数和约定必须沉淀到目标项目 wiki。

## 路由表

| 任务类型 | 指定角色 | 典型范围 |
| --- | --- | --- |
| 界面、HUD、菜单、提示、交互面板 | `ui-developer` | UI 结构、状态展示、输入反馈、界面绑定。 |
| 角色、镜头、输入、移动、手感 | `character-controller-developer` | Character、Camera、Control、动画状态接入。 |
| 关卡、场景、触发器、出生点、导航 | `scene-developer` | 场景对象、关卡流程、区域触发、场景交互。 |
| 攻击、受击、技能、伤害、状态 | `combat-developer` | 战斗判定、技能流程、连招、数值接入。 |
| 敌人、行为、感知、巡逻、决策 | `ai-developer` | 行为树、状态机、仇恨、目标选择。 |
| 背包、任务、存档、配置、事件流 | `gameplay-systems-developer` | 玩法系统、数据流、持久化、全局状态。 |
| 编辑器、批处理、导入导出、调试面板 | `tools-developer` | 开发工具、内容管线、验证辅助。 |
| 特效、材质、Shader、动画事件、表现接入 | `technical-art-developer` | 表现资源接入、特效触发、渲染相关配置。 |
| 复现、回归、验收、压力路径 | `qa-verification` | 验证清单、手动步骤、失败路径、回归风险。 |

## 分配规则

- 主程根据子任务主要交付物选择一个主角色。
- 子任务跨多个模块时，主程必须先定义接口契约，再拆成多个子任务。
- 同一文件同一时间只能分配给一个写入角色。
- QA 角色默认不参与所有任务；修复漏洞、多模块集成、核心玩法或高风险变更时必须考虑启用。
- 项目自定义职责只能写入项目 wiki，不能改写本通用角色定义。

## 读取规则

角色执行子任务时默认只读取：

1. 项目 wiki 子任务页。
2. 子任务页指定的项目 wiki 策划页面、任务页和接口契约。
3. 子任务指定的其他项目 wiki 页面。
4. 对应角色的 `DEFINITION.md` 和 `RULE.md`。
5. 必要的源码、资源或文档入口。

需要额外读取时，必须先说明原因、目标和预计影响，不能展开无关 wiki、历史记录或整目录。
