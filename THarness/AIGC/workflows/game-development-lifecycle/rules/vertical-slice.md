# 垂直切片规则

## 目标

用最小内容验证完整游戏结构能力，而不是第一轮填满完整内容规模。

## 最小切片必须覆盖

- 入口：主菜单、继续或新局入口。
- 选择：角色、难度、章节或等价起点。
- 推进：地图、关卡、回合、房间或等价节点。
- 核心玩法：至少一场完整战斗、解谜、经营或核心交互。
- 奖励：获得新能力、资源、卡牌、装备或等价成长。
- 构筑：玩家能调整构筑、装备、技能或策略。
- 变化节点：事件、商店、休整、锻造或等价非战斗节点。
- 结算：胜利、失败、章节结算和本局结算。

## 内容规模

第一轮内容只需要证明结构：

- 1 个可玩角色或等价起点。
- 1 条完整章节或等价流程。
- 1 个普通挑战、1 个变化节点、1 个终局挑战。
- 少量可验证的奖励和构筑项。

## 数据要求

- 所有内容对象都有稳定 ID。
- UI 文案、数值、素材引用和调试开关来自数据或配置。
- 引用关系可被脚本检查。
- 缺失素材和未实现效果有明确占位策略。

## 交付要求

- 玩家能从入口玩到结算。
- 开发者能通过 smoke 或等价脚本验证闭环。
- 失败路径不会卡死。
- 下一轮扩内容不需要重写核心结构。

## 禁止

- 禁止用大量内容掩盖核心循环未跑通。
- 禁止把策划完整度当作实现完成度。
- 禁止把一次性原型数据写死在 UI 或运行时代码里。
