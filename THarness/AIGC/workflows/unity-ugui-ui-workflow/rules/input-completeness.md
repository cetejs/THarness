# Unity UGUI UI 输入完整度规则

执行前必须检查：

- 最终效果图。
- 背景图、按钮素材、图标素材、装饰素材、Logo 等已切好的元素图片。
- 设计分辨率和目标屏幕比例。
- Unity 版本、Canvas Scaler 设置和旧版 `Text` 字体要求。
- 是否需要 Tween 库，以及已确认的 Tween 库名称。
- 是否需要自动生成 Prefab。
- 资源导入路径和输出路径。
- 交互说明、动态数据来源和界面跳转关系。
- 图片类型分类：最终效果图、纯背景图、按钮素材图、图标素材图、装饰素材图、Logo 图、Unity 实际截图。
- 字体资源状态：已提供字体、使用项目默认字体、需要替代字体、字体授权未知。
- 设计图比例与目标屏幕比例是否一致。
- 是否存在“最终效果图被误当正式资源”的风险。
- 是否存在“未切素材合集被误当正式元素资源”的风险。
- 是否触发上下文隔离：已切元素数量、图片总数、manifest 长度、是否需要生成 `.cs` 和 Prefab、截图校准轮次、交付报告长度。

## 分级规则

| 等级 | 判定 | 允许输出 | 阻塞内容 |
| --- | --- | --- | --- |
| L1 | 只有最终效果图 | 策划分析、元素需求清单、层级建议、第一版估算布局 | 完整代码、最终 Prefab、完整资源绑定、自动切图 |
| L2 | 效果图和已切元素素材 | L1 内容、资源验收记录、拼接还原文档、导入设置、资源依赖、施工方案 | 交互、动态数据、字体或状态未确认时不得写死 |
| L3 | 效果图、素材、项目设置和交互说明齐全 | 完整施工方案、集中配置、Editor 创建脚本、Controller、绑定表和运行检查 | 截图校准前不得宣称最终还原 |

## 图片分类要求

在进入任何阶段前，必须输出图片分类表：

| 图片 | 判断类型 | 是否可直接用于 Unity | 用途 | 风险 |
| --- | --- | --- | --- | --- |
| MainMenu_Final.png | 最终效果图 | 否 | 视觉参考 | 不能作为完整背景 |
| BG_MainMenu.png | 纯背景图 | 是 | 背景层 | 可整图使用 |
| Button_Start.png | 已切按钮元素图 | 是 | Button Sprite | 需审核状态和文字策略 |
| ButtonSheet.png | 未切素材合集 | 否 | 参考或要求用户补充元素图 | 当前工作流不自动切图 |

如果图片类型无法判断，必须标记为“待确认”，不能默认作为正式资源。

## 比例冲突规则

如果最终效果图比例与目标屏幕比例不同，必须输出：

1. 当前效果图比例。
2. 目标游戏比例。
3. 可能影响的区域。
4. 推荐处理方式：居中安全区、背景延展、两侧裁切、重做布局。

比例冲突未确认前，布局参数只能作为第一版估算值。

## 待确认规则

- 缺失项必须写入待确认项，不能编造。
- 可以估算的布局、字号、颜色或坐标必须标注“第一版估算值，需要 Unity 截图后校准”。
- 需要用户决定的技术路线、素材来源、字体、交互结果或业务跳转，不能由工作流默默选择。
- 如果输入不足以进入代码生成，只输出分析和交接包，不生成完整 Unity 脚本。
- 命中上下文隔离触发条件时，必须先选择快速路径、资源核查路径或独立施工路径，再决定是否文件化长产物。

## 成功标准

1. 当前输入等级明确。
2. 输出范围与输入等级一致。
3. 待确认项说明了影响和下一步处理方式。
4. 没有把估算值写成最终事实。
