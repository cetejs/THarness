# Unity UGUI UI 工作流

## 目标

把用户提供的 UI 完整展示图和已切好的元素图片，转成可在 Unity UGUI 中施工的交付物。默认采用瘦身路径：

```text
用户输入 UI 图片
-> UI 开发者判断输入等级和执行路径
-> UI 开发者完成界面语义、节点规划、施工方案和必要代码
-> 资源复杂时激活 UI 资源整理员做独立资源核查
-> Unity 截图校准并按问题回派修正
```

本工作流不再承担自动切图。用户必须提供完整展示图和已切好的元素图片；只有素材、分辨率、字体、交互、资源路径和项目设置齐全时，才进入 Prefab 与脚本施工。信息不足时，只输出估算值、缺失元素清单、待确认项和下一步输入要求。

## 执行路径

| 路径 | 触发条件 | 执行方式 | 输出 |
| --- | --- | --- | --- |
| 快速路径 | L1/L2 输入，资源数量少，用户未要求独立资源核查 | `ui-programmer` 在主会话或独立阶段完成输入判断、UI 规划、施工方案、草稿代码和校准摘要 | UI 规划、元素需求清单、布局估算、资源清单、施工方案或草稿脚本 |
| 资源核查路径 | 已切元素数量多、图片多、manifest 长、用户要求独立资源核查或 L3 完整交付 | `ui-programmer` 派发 `ui-asset-curator` 独立验收资源，只回收摘要和正式交接包 | 正式资源包、拼装还原文档、缺失/不可用资源清单、施工输入 |
| 独立施工路径 | 需要生成 `.cs` 和 Prefab 施工产物，且内容会明显膨胀 | `ui-programmer` 在独立窗口、subagent 或文件化阶段执行，不在 PL 主会话展开大段代码 | `UILayoutSpec`、Editor 创建脚本、Controller、绑定表、运行检查 |

## 任务类型判断

工作流激活后，必须先判断用户当前请求属于哪一种任务类型：

| 任务类型 | 触发语义 | 推荐路径 | 禁止事项 |
| --- | --- | --- | --- |
| 仅分析 UI | 分析这张图、这个界面是什么、有哪些元素 | 快速路径，只输出策划分析 | 不进入资源拼装和代码 |
| 制作施工方案 | 拆成 UGUI 层级、给布局表、给资源清单 | 快速路径；资源复杂时转资源核查路径 | 不使用未审核资源 |
| 资源拼装处理 | 验收用户已切元素、命名、分类、拼装还原 | 资源核查路径 | 不运行自动切图，不写 Controller |
| Unity 落地 | 生成 Prefab、Editor 脚本、Controller | 快速路径或独立施工路径 | 不跳过输入判断和资源门控 |
| 截图校准 | 对比目标图和 Unity 截图 | 快速路径；超过一轮时文件化偏差表 | 不修改 UI 功能语义 |
| 完整交付 | 从图片到施工报告和脚本 | 资源核查路径 + 独立施工路径 | 不把长表和长代码内联到主会话 |

## 激活条件

用户明确表达以下任一意图时激活：

- 根据 UI 截图、效果图或素材图生成 Unity UGUI 开发方案。
- 需要把 UI 完整展示图和已切元素图整理成 UGUI 节点层级、资源清单、布局表或 Prefab 施工输入。
- 需要生成 UGUI Editor Prefab 创建脚本、运行时 Controller 或截图校准反馈。
- 需要处理复杂 UI 的资源核查、文件化交付或多轮截图校准。

## 不适用条件

- 用户只要 UI 审美建议，不需要 Unity UGUI 落地。
- 用户指定 UI Toolkit、FairyGUI、Nova、NoesisGUI 或其他非 UGUI 技术路线。
- 用户没有提供完整展示图，也没有描述界面结构。此时必须先补充输入。
- 用户只提供未切素材合集，并要求本工作流自动切图。此时必须要求用户补充已切元素图，或明确切换到其他切图流程。

## 输入完整度分级

| 等级 | 输入 | 可交付 | 禁止承诺 |
| --- | --- | --- | --- |
| L1 | 只有最终效果图 | UI 规划、元素需求清单、UGUI 层级建议、RectTransform 第一版估算、待确认项 | 精准像素还原、完整交互逻辑、完整 Prefab 绑定、自动切图 |
| L2 | 效果图和已切元素素材 | L1 内容、资源验收记录、素材拼接还原文档、导入设置、资源路径依赖表、Prefab 生成方案、Controller 结构 | 未确认交互、动态数据、字体和按钮状态不得写死 |
| L3 | 效果图、素材、项目设置和交互说明 | L2 内容、集中布局配置、`CreateXXXPrefab.cs`、`XXXPanelController.cs`、绑定字段表、运行检查清单 | 不得绕过截图校准直接声明最终还原 |

## 默认读取

默认只读本文件和 `rules/INDEX.md`。按阶段继续读取：

1. 需要判断输入能做到哪一步时，读取 `rules/input-completeness.md`。
2. 需要输出 UI 语义、状态和交接输入时，读取 `rules/ui-planning.md`。
3. 需要验收已切元素、资源命名或拼接还原文档时，读取 `rules/technical-art-assembly.md`。
4. 需要生成 Unity C#、Prefab 创建脚本或 Controller 时，读取 `rules/unity-construction.md`。
5. 需要对比目标图和 Unity 截图并回派修正时，读取 `rules/screenshot-calibration.md`。
6. 触发复杂 UI、长 manifest、长报告、多轮校准或大代码产物时，读取 `rules/context-isolation.md`。
7. 需要交付报告结构时，读取 `templates/ui-ugui-delivery.md`。

## 角色边界

| 角色 | 职责 | 不负责 |
| --- | --- | --- |
| UI 开发者 `ui-programmer` | 输入等级判断、界面语义、玩家操作、元素需求、节点规划、资源核查调度、Prefab 施工方案、Editor 脚本、Controller、截图校准摘要和最终交付 | 自动切图、使用未审核资源、把复杂业务写进 Controller、在主会话内联长代码或长表 |
| UI 资源整理员 `ui-asset-curator` | 已切元素验收、资源分类、语义命名、`Final` / `ReferenceOnly` 划分、manifest 摘要、拼装还原文档、缺失和不可用资源清单 | 自动切图、重新设计功能、修改按钮含义、业务跳转逻辑、Unity C# 施工 |

## 会话接管与角色显示

- 工作流触发后，主会话默认接管为 `UI 开发者`，负责输入判断、路径选择、用户确认、阶段调度和最终交付。
- 每次阶段输出先声明角色，例如 `角色：UI 开发者`、`角色：UI 资源整理员`。
- 快速路径不强制启动资源 subagent；只有命中资源核查路径或上下文隔离触发条件时，才把资源验收交给 `ui-asset-curator` 独立执行。
- 需要生成 `.cs` 和 Prefab 施工产物时，不在 PL 主会话展开大段代码；应使用独立窗口、subagent 或文件化交付，并让主会话只保留摘要和验证结果。
- 用户指出角色、阶段或资源边界错误时，立即暂停当前阶段，回退到对应输入判断、资源核查或施工阶段补齐交接包。

## 上下文隔离策略

主会话只保留：

- 输入等级。
- 当前路径：快速路径 / 资源核查路径 / 独立施工路径。
- 阶段结论。
- 阻塞项。
- 用户需要决策的问题。

详细内容写入文件或独立产物：

- 资源验收文档。
- 拼装还原文档。
- Prefab 施工报告。
- 截图校准表。
- Controller / Editor 脚本文件。

详细触发条件见 `rules/context-isolation.md`。

## 执行阶段

| 阶段 | 输出 | 验证 |
| --- | --- | --- |
| 输入检查 | 输入完整度等级、执行路径和待确认项 | 不足信息没有被编造，估算值已标注。 |
| UI 规划 | UI 语义、元素需求、节点规划和资源核查需求 | 元素语义、状态、动态数据和按钮行为清楚。 |
| 资源核查 | `Final` 资源、`ReferenceOnly` 资源、manifest 摘要和拼装还原文档 | 正式资源可直接使用，缺失元素和不可用元素已标明。 |
| Unity 施工 | Prefab 施工方案、集中配置、Editor 脚本、Controller 和绑定表 | 每个 C# 文件只定义一个普通类；布局、资源路径和按钮配置集中。 |
| 截图校准 | 偏差表、修正参数和回派清单 | 目标图与 Unity 截图完成至少一轮对比；超过一轮时偏差表文件化。 |

## 阶段门控

- 输入等级和执行路径未明确时，不进入资源核查或施工。
- 资源核查路径中，`ui-asset-curator` 未输出可审核的已切元素资源包、拼装文档和资源可用性结论时，不进入 Unity 施工。
- 快速路径中，如果没有独立资源核查，`ui-programmer` 仍必须输出资源依赖表和未审核风险，不能把风险资源标为最终资源。
- 用户提供的已切元素作为正式资源时，正式资源集合必须全部可直接使用；不可直接使用的污染元素、未切素材合集或参考图必须放入 `ReferenceOnly`，不得进入施工资源表。
- 交付报告超过 200 行、资源清单很长、manifest 很长或截图校准超过 1 轮时，详细表格写文件，主会话只读摘要。

## 完成标准

- 输入等级、执行路径和上下文隔离判断清楚。
- 简单 UI 没有被强制拆成多角色流程。
- 复杂 UI 的资源、代码、截图校准和长报告已文件化或独立上下文处理。
- 所有可交互元素都能追溯到按钮行为说明。
- 所有 Image 都能追溯到已审核元素资源路径或占位路径。
- 正式施工资源已标记为可直接使用；不可直接使用的资源没有进入 Unity 施工依赖。
- 所有旧版 `Text` 文本都标明固定文本或动态数据来源。
- 所有布局参数来自布局表或集中配置。
- 生成脚本符合 Unity 编译要求和每文件单类规则。
- 截图校准有明确回传格式、回派角色和下一轮修正表。
