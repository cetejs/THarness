# 模块边界

## 状态

- status: draft
- source: 当前方案判断
- updated: 2026-05-15

## 结论

WorldSeed 至少需要区分项目治理层、生成器层、Godot 运行时层、资产管线和验证层。

## 初始边界

| 模块 | 职责 | 不负责 |
| --- | --- | --- |
| 项目适配层 | 保存项目事实、任务、决策和运行记录 | 运行游戏逻辑 |
| GameSpec | 保存结构化游戏规格 | 直接操作 Godot 文件 |
| 生成器 | 把 GameSpec 转成 Godot 工程内容 | 决定产品需求 |
| Godot 运行时 | 执行生成出的游戏 | 解释用户对话 |
| 资产管线 | 生成、导入、绑定和校验资源 | 改写玩法规则 |
| 验证器 | 运行 Godot、截图、模拟输入和检查资源 | 生成新功能 |

## 验证

开发任务必须指定主要模块和禁止修改范围。
