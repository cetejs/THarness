# 架构索引

| 文件 | read_when |
| --- | --- |
| `product-vision.md` | 需要了解产品级架构目标。 |
| `godot-runtime-boundary.md` | 需要判断 Godot 引擎边界、插件边界或运行时边界。 |
| `ai-generation-pipeline.md` | 需要了解从对话到可玩游戏的生成链路。 |
| `game-spec-schema.md` | 需要定义或修改 GameSpec 中间表示。 |
| `module-boundaries.md` | 需要判断 UI、玩法、数值、音频、动画、生成器和验证器边界。 |
| `configuration-surface.md` | 需要判断参数、路径、模型、导出选项和生成策略放在哪里。 |

## 规则

- 架构页面只保存稳定结构和边界。
- 未确认技术选择写入 `../open-questions.md`。
- 具体开发任务写入 `../development/tasks/`。
