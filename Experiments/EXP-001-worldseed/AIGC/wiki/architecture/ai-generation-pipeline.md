# AI 生成链路

## 状态

- status: draft
- source: 用户确认和当前方案判断
- updated: 2026-05-15

## 结论

生成链路应避免让 AI 直接自由写完整游戏工程，优先采用结构化 GameSpec 驱动生成。

## 推荐链路

```text
用户对话
-> 需求澄清
-> GameSpec
-> 生成计划
-> Godot 工程生成
-> 资产生成或占位资源绑定
-> Godot 自动运行验证
-> 可玩交付
```

## 约束

- LLM 输出必须先进入可校验结构。
- Godot 代码生成应来自模板、规则和 schema，而不是无约束自由输出。
- 资源缺失时必须使用可追踪占位资源或明确失败。

## 验证

后续生成器任务必须包含 GameSpec 校验和 Godot smoke test。
