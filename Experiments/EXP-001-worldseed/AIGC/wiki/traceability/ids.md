# ID 规则

## 状态

- status: active
- source: 用户确认和当前项目适配层规划
- updated: 2026-05-15

## ID 类型

| 类型 | 格式 | 用途 |
| --- | --- | --- |
| 需求 | `REQ-YYYYMMDD-001` | 用户确认的需求。 |
| 决策 | `DEC-YYYYMMDD-001` | 产品或技术决策。 |
| 实验 | `EXP-YYYYMMDD-001-short-name` | 实验计划、执行和结果。 |
| 开发任务 | `TASK-YYYYMMDD-001` | 主开发任务。 |
| 子任务 | `SUB-YYYYMMDD-001` | 模块子任务。 |
| 变更 | `CHG-YYYYMMDD-001-short-name` | 正式新增、修改或删除记录。 |
| 验证 | `VER-YYYYMMDD-001-short-name` | 测试、检查或人工验证记录。 |

## 编号规则

- 日期使用当前日期。
- 同一天同类型从 `001` 递增。
- 短名称使用英文小写和连字符。
- ID 一经写入正式记录，不复用、不改名。
- 创建新 ID 前，必须同时查看 `records/INDEX.md` 和记录目录中的现有文件，按同日同类型最大编号顺延。
- 中间编号空缺、文件未提交或旧文件删除，都不能作为复用编号的理由。
- 记录文件名必须以完整 ID 开头，一级标题、文件名和索引中的 ID 必须完全一致。
