# 主界面 UI 界面开发规格案

## 状态

- status: active
- source: `migration-gdd/screens/main-menu.md`; `ui-effect-image-gallery.md`; 用户确认的主界面 UI 开发策划拆解口径

## 结论

主界面施工按最小拆分执行：一张全屏背景、一组 Logo、四个按钮和一个版本号文本。静态森林、花境、法阵、五行符号、藤蔓、花朵和竖排装饰文案全部并入背景，不单独创建运行时装饰图层。

## 4. 布局合同

坐标原点为左上角，单位为像素，目标分辨率为 `1920x1080`。

本规格案中的 `x / y / w / h` 永远表示目标分辨率下最终渲染矩形的左上角坐标和尺寸，不是 Unity `RectTransform.anchoredPosition` 或其他引擎的本地锚点坐标。锚点和 Pivot 只用于适配换算；程序实现必须先保证最终渲染矩形与本表一致，再按目标 UI 系统换算本地坐标。

| `main-menu-logo-group` | 483 | 150 | 974 | 270 | Top-Center | 0.5,0.5 | 10 | 不允许裁切；不随文本变化。 |

## 8. 状态矩阵

| 状态 | 触发条件 | 画面变化 | 可交互变化 | 文案或提示 |
| --- | --- | --- | --- | --- |
| 默认 | 主界面打开且数据加载完成 | 背景、Logo、四按钮、版本号显示 | 可用按钮接收点击 | 无 |
| 悬停/聚焦 | 鼠标悬停或手柄焦点落在按钮上 | 按钮底板亮度提升或描边增强，不改变尺寸 | 当前按钮可确认 | 无 |
| 按下 | 鼠标按下或确认键按下 | 按钮底板切 pressed 态，整体缩放至 98% | 释放后触发事件 | 无 |
| 禁用 | 对应 `can*` 字段为 false 或系统未就绪 | 按钮使用 disabled 态，文本透明度降低 | 禁止点击触发业务 | 按钮级失败提示按控件合同执行。 |
| 返回主界面 | 从图鉴或设置关闭 | 主界面恢复默认状态 | 四按钮重新接收输入 | 无 |
| 错误 | 入口点击时系统拒绝 | 不跳转，按钮恢复默认或禁用态 | 仍停留主界面 | 显示对应失败提示。 |

## 10. 一致性验收合同

本节只定义主界面 UI 的验收口径，不替代工程自动化测试方案。

| 区域 | 验收方式 | 通过标准 |
| --- | --- | --- |
| 全屏截图 | 在 `1920x1080` 视口截图 | 背景覆盖全屏，无黑边，无 UI 元素缺失。 |
| Logo 组 | 对比 `1920x1080` 截图 | 位置误差不超过 `±6px`，尺寸误差不超过 `±6px`。 |
| 按钮组 | 对比 `1920x1080` 截图 | 每个按钮位置误差不超过 `±4px`，尺寸误差不超过 `±4px`，四按钮垂直顺序正确。 |
| 按钮文字 | 截图人工检查 | 文本居中，完整可读，不溢出按钮底板。 |
| 版本号 | 对比 `1920x1080` 截图 | 右下角位置误差不超过 `±6px`，文本完整可读。 |
| 背景 | 截图人工检查 | 当前按“可验收一致”执行：中央法阵、左右花境、暗角构图一致；不要求像素级一致。若后续确认“一模一样”，必须补充正式背景图和关键区域像素误差阈值。 |
| 按钮交互 | 鼠标或触摸点击 | hitbox 覆盖完整按钮底板；快速连点开始按钮只触发一次进入开局界面。 |

## 11. 不可接受差异

- 背景存在黑边、拉伸变形或核心法阵被裁切。
- Logo 组与按钮组重叠。
- 任一按钮文字烘焙在背景图或按钮底板图中。
- 版本号烘焙在背景图中。
- 四个按钮顺序被改变。
- 禁用按钮仍然触发跳转。
- 快速连点开始按钮导致重复创建开局流程。

## 验证

本规格案依据主界面迁移策划案、主界面参考图和用户确认的 UI 开发策划拆解口径创建。后续施工前应由程序和美术共同确认未决问题，并按“一致性验收合同”执行截图和交互验证。
