# UI 策划案验收清单

## 状态

- status: active
- source: 用户确认的实验二目标
- updated: 2026-05-17

## 结论

UI 策划案只有通过本清单，才能进入“交给程序按案开发”的验证阶段。未通过项不是程序问题，而是策划输入不足。

## 基础门槛

| 检查项 | 通过标准 |
| --- | --- |
| 效果图明确 | 有路径、分辨率和目标视口。 |
| 界面目标明确 | 玩家在该界面的目标可以一句话说明。 |
| 来源去向明确 | 进入来源、成功去向、取消去向和失败路径已写清。 |
| 层级已拆解 | 效果图中的主要视觉区域都被归类为背景、装饰、内容、交互或覆盖层。 |
| 动态内容已标注 | 文本、数字、图标、列表和状态变化都有数据来源或明确占位规则。 |
| 素材边界明确 | 效果图、参考图、可用运行时素材、需切片素材和缺失素材没有混用。 |

## 布局检查

| 检查项 | 通过标准 |
| --- | --- |
| 区域位置可执行 | 每个主要区域有位置规则、尺寸或占屏比例。 |
| 对齐和间距明确 | 不依赖“看起来差不多”判断。 |
| 层级顺序明确 | 背景、装饰、内容、按钮、弹窗和遮罩不会互相抢层级。 |
| 溢出策略明确 | 长文本、列表过长和窗口缩放后的处理已定义。 |
| 不裁切区域明确 | 核心信息和关键按钮在验收视口内必须完整可见。 |

## 交互检查

| 检查项 | 通过标准 |
| --- | --- |
| 控件状态完整 | 默认、选中、按下、禁用至少有规则。 |
| 点击反馈明确 | 点击、不可点击和失败反馈已定义。 |
| 跳转结果明确 | 每个关键操作都有去向或事件。 |
| hitbox 明确 | 关键控件有最小点击范围或点击区域规则。 |
| 快速操作可验收 | 快速连点、返回、重复进入不会产生未定义状态。 |

## 一致性检查

| 检查项 | 通过标准 |
| --- | --- |
| 严格还原区域已标注 | 必须接近效果图的区域有容差或判断方式。 |
| 动态适配区域已标注 | 可以因数据或分辨率变化而调整的区域有规则。 |
| 不可接受差异已列出 | 程序实现后不能出现的偏差已明确。 |
| 验收截图明确 | 需要在哪些视口截图验收已明确。 |
| 归因方式明确 | 差异能归因为策划允许、素材缺失、实现问题或引擎限制。 |

## 判定

| 结果 | 含义 |
| --- | --- |
| 通过 | 可以交给程序进行界面实现验证。 |
| 有条件通过 | 可以进入实现，但未决问题必须登记并限制实现范围。 |
| 不通过 | 不应进入实现，应先补策划案。 |
