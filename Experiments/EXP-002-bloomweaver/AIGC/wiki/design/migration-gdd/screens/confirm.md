# 确认弹窗迁移策划案

## 定位

确认弹窗用于处理高风险操作，例如返回主界面、重新开始、退出、重置设置、消耗稀有资源。它是通用模态组件。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `confirm-screen` |
| 参考图 | `images/ui-reference-20260517/confirm.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/confirm-screen.prefab` |
| 目标技术 | 新引擎 UI 弹窗 |
| 显示层级 | 模态覆盖层 |
| 来源阶段 | 任意高风险操作 |
| 退出方向 | 来源界面或确认动作目标 |

## 玩家目标

- 理解即将执行的操作。
- 选择取消或确认。
- 避免误触导致丢失进度。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 背景 | 半透明暗色遮罩，阻止下层输入。 |
| 中央 | 标题、说明、影响预览。 |
| 底部 | 取消、确认。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `title` | string | 调用方 | 弹窗标题。 |
| `message` | string | 调用方 | 说明文本。 |
| `preview` | string | 调用方 | 影响预览。 |
| `confirmStyle` | enum | 调用方 | 普通或危险样式。 |
| `canAccept` | bool | 调用方 | 确认按钮状态。 |
| `onAccept` | callback | 调用方 | 确认动作。 |
| `onCancel` | callback | 调用方 | 取消动作。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 取消 | 点击 | 关闭弹窗，执行取消回调。 |
| 确认 | 点击 | 锁定按钮，执行确认回调，完成后关闭或跳转。 |
| 返回键 | 按下 | 等同取消。 |

## 无效输入

- 确认回调执行中不能重复点击。
- 点击遮罩外不执行确认动作。
- 缺少确认回调时确认按钮禁用或只关闭弹窗，不能报错。

## 迁移实现要点

- 确认弹窗应由全局弹窗服务创建和销毁。
- 下层界面输入必须被遮罩阻断。
- 危险操作按钮使用独立样式。

## 迁移验收

- 打开弹窗后下层不可点击。
- 确认回调只执行一次。
- 取消不会执行确认动作。
- 返回键关闭弹窗并回来源界面。
