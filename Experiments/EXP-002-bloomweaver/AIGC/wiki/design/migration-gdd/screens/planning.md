# 布阵界面迁移策划案

## 定位

布阵界面是本作最核心的玩家操作界面。玩家在这里观察 12x6 灵枢盘，连接节点形成灵脉，为接下来的自动战斗预埋触发条件。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `planning-screen` |
| 参考图 | `images/ui-reference-20260517/planning.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/planning-screen.prefab` |
| 目标技术 | UI + 棋盘交互层 |
| 显示层级 | 主页面 |
| 来源阶段 | 开局界面、商店界面、下一波开始 |
| 退出方向 | 战斗界面、阵图总览、主界面确认 |

## 玩家目标

- 判断当前棋盘的五行和特殊节点分布。
- 绘制合法灵脉。
- 在长灵脉爆发和短灵脉稳定之间做取舍。
- 查看本波威胁并决定是否开始演武。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 阶段、波次、目标、资源、提示。 |
| 左侧 | 5 名花灵状态、属性、职业和生命。 |
| 中央 | 灵枢盘、节点、灵脉线、扫描预览。 |
| 右侧 | 本波敌潮威胁、推荐应对。 |
| 底部 | 撤销、清除此脉、重织、开始演武。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `waveIndex` | int | 流程系统 | 波次显示。 |
| `boardNodes` | 12x6 list | 棋盘系统 | 节点显示和输入。 |
| `confirmedLines` | list | 连线系统 | 已确认灵脉。 |
| `currentLine` | list | 连线系统 | 当前拖拽灵脉。 |
| `occupiedNodes` | set | 连线系统 | 禁止重复占用。 |
| `partyStatus` | list | 阵容/敌人系统 | 左侧花灵状态。 |
| `waveThreat` | object | 波次系统 | 右侧威胁预览。 |
| `canStartCombat` | bool | 开战规则 | 开始按钮状态。 |

## 交互规格

| 操作 | 规则 | 结果 |
| --- | --- | --- |
| 按下节点 | 节点充盈且未被占用 | 开始绘制灵脉。 |
| 拖到相邻节点 | 必须四向相邻、未占用、不交叉 | 加入当前灵脉。 |
| 拖回已有节点 | 允许回退 | 截断后续节点。 |
| 松开鼠标或触摸 | 当前灵脉长度大于等于 2 | 确认灵脉。 |
| 撤销 | 存在已确认灵脉 | 删除最后一条灵脉。 |
| 清除此脉 | 当前有正在绘制或选中灵脉 | 清除对应灵脉。 |
| 重织 | 有次数或资源 | 重新生成棋盘。 |
| 开始演武 | 满足开战条件 | 进入战斗。 |

## 无效输入

- 非相邻节点不能连接。
- 已占用节点不能连接。
- 当前线自交时拒绝。
- 穿过已确认线段时拒绝。
- 节点状态不是充盈时拒绝。
- 空棋盘或阵容缺失时不能开始演武。

## 迁移实现要点

- 棋盘坐标和屏幕坐标转换要独立封装。
- 连线合法性必须在玩法层实现，不能只靠 UI 阻止。
- 棋盘节点、灵脉线、扫描预览要支持引擎替换，例如 UI 控件、2D 节点或自绘网格。
- 视觉拒绝反馈和规则校验分开，避免视觉状态决定逻辑结果。

## 迁移验收

- 能在 12x6 棋盘上绘制合法灵脉。
- 非法连线全部被阻止。
- 已确认灵脉能被战斗扫描系统读取。
- 撤销和清除后占用缓存正确刷新。
- 点击开始演武后不再允许继续布阵。
