# 三选一奖励界面迁移策划案

## 定位

三选一奖励界面是波间构筑的核心决策。玩家在每波胜利后从 3 个造化中选择 1 个，让本局打法逐渐成型。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `reward-screen` |
| 参考图 | `images/ui-reference-20260517/reward.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/reward-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 波次结算点击进入造化 |
| 退出方向 | 商店界面、阵图总览 |

## 玩家目标

- 比较 3 个奖励。
- 判断奖励与当前构筑是否匹配。
- 选择一个奖励进入商店。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 标题、当前波次、资源。 |
| 中央 | 3 张奖励卡，横向排列。 |
| 底部 | 当前构筑预览、选择说明、确认按钮。 |
| 辅助 | 跳过、阵图总览。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `rewardChoices` | list | 奖励系统 | 3 张卡数据。 |
| `selectedRewardIndex` | int | UI 状态 | 当前选中卡。 |
| `activeRewards` | list | 运行时属性 | 当前构筑摘要。 |
| `rewardPreview` | object | 构筑分析 | 选择后变化预览。 |
| `spiritSand` | int | 经济系统 | 顶部资源。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 奖励卡 | 点击 | 高亮卡片，刷新选择预览。 |
| 确认选择 | 点击 | 应用奖励，只应用一次，进入商店。 |
| 跳过 | 点击 | 按规则跳过或给补偿，进入商店。 |
| 阵图总览 | 点击 | 打开总览覆盖层，关闭后回奖励。 |

## 无效输入

- 未选择奖励时确认按钮禁用。
- 奖励应用中按钮锁定，防止重复应用。
- 奖励池为空时显示空状态，不进入死流程。

## 迁移实现要点

- 奖励抽取由奖励系统完成，界面只展示结果。
- 奖励应用必须写入运行时属性，不修改原始配置。
- 进入商店是奖励选择后的默认流程。
- 奖励卡要支持类型、名称、描述、数值、适配建议。

## 迁移验收

- 每次打开奖励界面显示 3 个不重复合法奖励，奖励池不足时按池大小显示。
- 选中奖励后预览刷新。
- 确认后奖励只生效一次。
- 确认后进入商店界面。
