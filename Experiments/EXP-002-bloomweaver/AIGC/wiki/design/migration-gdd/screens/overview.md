# 阵图总览界面迁移策划案

## 定位

阵图总览界面用于让玩家查看当前构筑全貌。它可以从布阵、暂停、奖励、商店和结算打开，关闭后必须回到来源界面。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `overview-screen` |
| 参考图 | `images/ui-reference-20260517/overview.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/overview-screen.prefab` |
| 目标技术 | 新引擎 UI 覆盖层 |
| 显示层级 | 覆盖层 |
| 来源阶段 | 布阵、暂停、奖励、商店、结算 |
| 退出方向 | 来源界面、布阵界面、总结界面 |

## 玩家目标

- 查看完整阵图和灵脉分布。
- 查看花灵、造化、敌潮、统计等页签。
- 判断当前构筑优势和短板。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 标签页：阵图、花灵、造化、敌潮、统计。 |
| 中央 | 当前阵图或对应标签主内容。 |
| 左侧 | 构筑摘要、优势、关键节点。 |
| 右侧 | 风险、敌潮压力、状态总览。 |
| 底部 | 关闭、返回观阵、本局总结。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `sourceScreenKey` | string | UI 导航 | 关闭后返回。 |
| `selectedTab` | enum | UI 状态 | 当前标签。 |
| `boardSnapshot` | object | 棋盘/连线系统 | 阵图显示。 |
| `partySnapshot` | list | 阵容系统 | 花灵标签。 |
| `activeRewards` | list | 奖励系统 | 造化标签。 |
| `enemyThreat` | object | 波次系统 | 敌潮标签。 |
| `runStats` | object | 战斗统计 | 统计标签。 |
| `buildAnalysis` | object | 构筑分析 | 优势短板。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 标签按钮 | 点击 | 切换中央内容和左右摘要。 |
| 关闭 | 点击 | 返回来源界面。 |
| 返回观阵 | 点击 | 若规则允许，进入布阵界面。 |
| 本局总结 | 点击 | 若处于结算后，打开总结或胜利界面。 |

## 无效输入

- 战斗中不能直接返回观阵，除非未来加入特殊规则。
- 来源界面丢失时默认回主流程当前阶段界面。
- 没有数据的标签显示空状态，而不是空白。

## 迁移实现要点

- 总览展示应读取快照或只读 ViewModel，不能修改棋盘和奖励。
- 来源返回必须由 UI 导航栈记录。
- 战斗中打开总览时默认展示快照，避免实时变化导致阅读困难。

## 迁移验收

- 从布阵打开总览，关闭后回布阵。
- 从暂停打开总览，关闭后回暂停，战斗仍暂停。
- 标签切换内容正确。
- 不允许返回观阵时按钮禁用并提示。
