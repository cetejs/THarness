# 局外养成界面迁移策划案

## 定位

局外养成界面是长期成长入口。当前项目尚未实现完整局外成长，因此迁移首版可以做成可打开的占位系统，后续再接真实节点树和资源。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `garden-screen` |
| 参考图 | `images/ui-reference-20260517/garden.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/garden-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面或覆盖层 |
| 来源阶段 | 胜利结算、花灵图鉴、主界面扩展入口 |
| 退出方向 | 来源界面 |

## 玩家目标

- 查看局外成长节点。
- 了解可解锁内容和消耗。
- 消耗局外资源解锁成长。
- 返回来源界面。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 左侧 | 分类菜单：花灵唤醒、灵枢修复、造化研究、灵市扩建、敌潮研究。 |
| 中央 | 节点树。 |
| 右侧 | 选中节点详情、效果、消耗、前置条件。 |
| 底部 | 局外资源栏和返回按钮。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `gardenCategories` | list | 配置 | 左侧分类。 |
| `selectedCategory` | enum | UI 状态 | 当前分类。 |
| `gardenNodes` | list | 局外成长配置 | 节点树。 |
| `selectedNodeId` | string | UI 状态 | 当前节点。 |
| `metaResources` | object | 存档系统 | 局外资源。 |
| `unlockState` | object | 存档系统 | 节点状态。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 分类菜单 | 点击 | 切换节点树。 |
| 节点 | 点击 | 显示节点详情。 |
| 解锁 | 点击 | 校验前置和资源，成功后解锁。 |
| 返回 | 点击 | 回来源界面。 |

## 无效输入

- 前置未满足时解锁按钮禁用。
- 资源不足时解锁按钮禁用或点击提示。
- 节点满级时显示已满级。
- 局外系统未开放时显示“暂未开放”，不报错。

## 迁移实现要点

- 局外资源必须和单局灵砂区分。
- 节点效果要配置化，不能硬编码在 UI。
- 如果首版不做真实局外成长，仍需要保留界面入口状态和明确提示。

## 迁移验收

- 能打开和关闭局外养成界面。
- 点击分类和节点能刷新右侧详情。
- 解锁校验不会错误扣资源。
- 解锁状态能持久化。
