# 胜利结算界面迁移策划案

## 定位

胜利结算界面用于展示最终守阵成功。它是单局完成后的情绪高点，要明确展示总评级、总收益和下一步出口。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `victory-screen` |
| 参考图 | `images/ui-reference-20260517/victory.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/victory-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 达成目标波次 |
| 退出方向 | 主界面、局外养成、开局界面 |

## 玩家目标

- 查看最终评级和总统计。
- 确认本局收获。
- 选择再来一局、返回主界面或进入局外养成。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 胜利标题，例如“灵圃复苏”。 |
| 中央 | 总评级、总波数、总击杀、总突破、存活、总时长。 |
| 中下 | 本局收获、造化数量、商店购买、最终构筑摘要。 |
| 底部 | 返回主界面、前往万花残庭、再来一局。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `targetWave` | int | 流程配置 | 目标波数。 |
| `finalWave` | int | 流程系统 | 实际完成波数。 |
| `killCount` | int | 战斗统计 | 总击杀。 |
| `breachCount` | int | 战斗统计 | 总突破。 |
| `aliveSpiritCount` | int | 阵容系统 | 存活花灵。 |
| `duration` | float | 流程系统 | 总时长。 |
| `totalGold` | int | 经济系统 | 总灵砂收益。 |
| `totalExp` | int | 经济系统 | 总识海收益。 |
| `activeRewardCount` | int | 奖励系统 | 造化数量。 |
| `shopPurchaseCount` | int | 商店系统 | 商店购买次数。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 返回主界面 | 点击 | 结束本局，回主界面。 |
| 前往万花残庭 | 点击 | 打开局外养成界面。 |
| 再来一局 | 点击 | 开启新局，进入开局界面。 |

## 无效输入

- 局外养成未开放时按钮禁用并提示。
- 胜利奖励入账必须只执行一次。
- 快速连点再来一局不能创建多份新局状态。

## 迁移实现要点

- 胜利结算使用整局快照，不从已清理对象读取。
- 最终评级建议先按突破次数和存活花灵计算，后续再扩展。
- 局外资源和单局资源要分开，避免把灵砂误当作局外货币。

## 迁移验收

- 达成目标波次后进入胜利结算。
- 总统计与本局过程一致。
- 再来一局能重置局内状态。
- 返回主界面后无旧局残留。
- 局外养成未实现时有明确禁用或提示。
