# 主界面迁移策划案

## 定位

主界面是玩家进入游戏后的第一视觉锚点，负责传达东方五行花境题材，并提供新局入口、图鉴入口、设置入口和退出入口。迁移时必须保证它是“游戏入口”，不是宣传页。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `main-menu-screen` |
| 参考图 | `images/ui-reference-20260517/main-menu.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/main-menu-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 应用启动、返回主界面 |
| 退出方向 | 开局界面、花灵图鉴、设置、退出应用 |

## 玩家目标

- 快速开始一局守阵。
- 查看花灵资料。
- 调整设置。
- 退出游戏。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 背景 | 幽暗森林和花境入口，强调神秘、宁静、即将启阵。 |
| 顶部中心 | 游戏 Logo 和副标题。 |
| 中央 | 四个纵向主按钮。 |
| 左右边缘 | 五行符号、竖排装饰和花纹。 |
| 右下角 | Demo 版本号。 |

按钮顺序固定为：

1. 开始守阵。
2. 花灵图鉴。
3. 设置。
4. 退出。

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `title` | string | 本地配置 | 游戏标题。 |
| `subtitle` | string | 本地配置 | 副标题或版本标识。 |
| `version` | string | 构建配置 | 版本号。 |
| `canStartRun` | bool | 流程系统 | 开始按钮是否可用。 |
| `canOpenCodex` | bool | 图鉴系统 | 图鉴按钮是否可用。 |
| `canExit` | bool | 平台能力 | 退出按钮是否可用。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| `main-menu-start-button` | 点击 | 进入开局界面。 |
| `main-menu-codex-button` | 点击 | 打开花灵图鉴，关闭后回主界面。 |
| `main-menu-settings-button` | 点击 | 打开设置界面，关闭后回主界面。 |
| `main-menu-exit-button` | 点击 | 退出应用或弹出退出确认。 |

## 无效输入

- 若新局初始化资源未加载完成，开始按钮禁用并提示“资源加载中”。
- 若图鉴系统未接入，花灵图鉴按钮禁用或点击后显示“图鉴暂未开放”。
- 退出在编辑器或调试环境中不得导致异常，应走平台适配接口。

## 迁移实现要点

- 主界面只派发导航命令，不直接创建战斗对象。
- 背景、Logo、按钮底板、装饰必须拆成独立资源，不把动态文字烘焙进背景。
- 按钮反馈统一使用默认、聚焦、按下、禁用四种状态。
- 主界面恢复时需要清理上一局残留的战斗对象和覆盖层。

## 迁移验收

- 点击开始守阵只进入一次开局界面。
- 从图鉴、设置关闭后能回到主界面。
- 快速连点开始按钮不会重复创建新局。
- 版本号能通过配置或构建信息刷新。
- 未开放入口有禁用状态或明确提示。
