# 暂停界面迁移策划案

## 定位

暂停界面用于战斗中断、状态查看和高风险操作。它必须是覆盖层，关闭后返回战斗或指定页面，不能破坏当前战斗状态。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `pause-screen` |
| 参考图 | `images/ui-reference-20260517/pause.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/pause-screen.prefab` |
| 目标技术 | 新引擎 UI 覆盖层 |
| 显示层级 | 覆盖层 |
| 来源阶段 | 战斗界面 |
| 退出方向 | 战斗界面、阵图总览、设置、主界面 |

## 玩家目标

- 临时暂停战斗。
- 查看当前战况和花灵状态。
- 打开设置。
- 重新开始或返回主界面。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 背景 | 战斗画面冻结，加暗色遮罩。 |
| 中央 | 暂停标题和当前战况摘要。 |
| 左侧或中部 | 花灵状态、波次、击杀、突破、存活。 |
| 右侧 | 纵向菜单按钮。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `combatSnapshot` | object | 战斗统计 | 当前战况摘要。 |
| `partyStatus` | list | 敌人/阵容系统 | 花灵生命和状态。 |
| `sourceScreenKey` | string | UI 导航 | 返回来源。 |
| `canResume` | bool | 流程系统 | 继续按钮状态。 |
| `canRestart` | bool | 流程系统 | 重开按钮状态。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 继续演武 | 点击 | 关闭暂停并恢复战斗时间。 |
| 阵图总览 | 点击 | 打开总览覆盖层，关闭后回暂停。 |
| 花灵状态 | 点击 | 打开花灵图鉴或状态页，关闭后回暂停。 |
| 设置 | 点击 | 打开设置覆盖层，关闭后回暂停。 |
| 重新开始本局 | 点击 | 打开确认弹窗，确认后重开。 |
| 返回主界面 | 点击 | 打开确认弹窗，确认后清理战斗并回主界面。 |

## 无效输入

- 非战斗阶段不能打开暂停界面。
- 重开或返回主界面执行中，所有暂停按钮锁定。
- 打开设置或总览后，下层暂停按钮不可误触。

## 迁移实现要点

- 暂停必须控制统一时间源，而不是逐个暂停对象。
- 覆盖层关闭要根据 `sourceScreenKey` 返回，不直接恢复战斗。
- 重新开始和返回主界面必须通过流程系统清理运行时对象。

## 迁移验收

- 暂停后敌人、投射体、倒计时和伤害数字全部停止。
- 继续后战斗从暂停前状态恢复。
- 从暂停打开总览，关闭后仍回暂停。
- 重开和返回主界面不会留下旧敌人或旧灵脉。
