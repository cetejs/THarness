# 战斗界面迁移策划案

## 定位

战斗界面展示玩家布阵结果。玩家不直接点击攻击，而是观察花灵自动扫描棋盘、触发灵脉、攻击敌潮。迁移时必须保留“战斗主体表现层”和“HUD 信息层”的分离。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `combat-screen` |
| 参考图 | `images/ui-reference-20260517/combat.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Battle2D/combat-screen.prefab` |
| 目标技术 | 2D 战斗场景 + UI HUD |
| 显示层级 | 主页面 |
| 来源阶段 | 布阵界面点击开始演武 |
| 退出方向 | 暂停界面、波次结算、失败结算、胜利结算 |

## 玩家目标

- 观察灵脉是否被频繁命中。
- 识别当前输出、治疗和防线压力。
- 在危险时暂停查看状态。
- 等待本波结束或失败。

## 视觉与布局

| 层级 | 内容 |
| --- | --- |
| 战场背景 | 森林、路径、灵圃地面。 |
| 花灵层 | 5 名上阵花灵，显示攻击、受击、倒下状态。 |
| 敌人层 | 普通敌人、精英、Boss，沿路径推进。 |
| 棋盘层 | 当前灵枢盘、灵脉线、枯竭和复苏状态。 |
| 特效层 | 投射体、命中、燃烧、护盾、治疗、必杀震动。 |
| HUD 层 | 波次、计时、资源、击杀、突破、暂停。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `waveIndex` | int | 流程系统 | HUD 波次。 |
| `timerSeconds` | float | 流程系统 | HUD 倒计时。 |
| `partyRuntime` | list | 战斗/敌人系统 | 花灵生命、攻速、等级。 |
| `enemies` | list | 敌人系统 | 敌人生成、位置、生命、属性。 |
| `confirmedLines` | list | 连线系统 | 扫描命中和灵脉状态。 |
| `scanEvents` | event | 扫描系统 | 扫描框显示。 |
| `attackResults` | event | 结算系统 | 投射体、伤害、治疗、护盾。 |
| `combatStats` | object | 战斗统计 | 击杀、突破、资源。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 暂停按钮 | 点击 | 暂停所有战斗时间，打开暂停界面。 |
| 敌人或花灵悬停 | 可选 | 显示基础信息，不影响战斗。 |
| 快捷键暂停 | 可选 | 等同暂停按钮。 |

战斗自动行为：

- 敌人按波次配置生成并推进。
- 花灵在敌人进入范围后按攻速计时。
- 花灵使用预生成扫描中心执行扫描。
- 扫描命中后由结算系统产出攻击结果。
- 攻击结果驱动投射体、伤害、治疗、护盾和飘字。
- 枯竭灵脉按复苏计时或治疗职业效果恢复。

## 无效输入

- 战斗结束后暂停按钮禁用。
- 暂停期间敌人、投射体、计时、扫描和掉落必须停止。
- 结算跳转期间所有 HUD 按钮锁定，避免重复进入结算。

## 迁移实现要点

- 战斗逻辑使用固定更新或清晰的时间源，暂停时统一停止。
- 视觉投射体可以延迟应用伤害，但伤害回调必须保证目标仍存活。
- 花灵倒下后停止对应花灵攻击计时。
- 所有战斗事件应可被 UI 和特效层订阅，但 UI 不应直接计算伤害。
- 战斗和 HUD 分层，方便新引擎替换渲染方案。

## 迁移验收

- 敌人能生成、推进、受击、死亡和突破。
- 花灵能自动扫描并触发普攻、技能、必杀。
- 灵脉命中后变枯竭，复苏后可再次触发。
- 暂停后所有战斗时间停止。
- 胜利或失败只触发一次结算跳转。
