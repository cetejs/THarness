# 波次结算界面迁移策划案

## 定位

波次结算界面用于反馈当前波次守阵结果，并承接进入造化奖励。它让玩家理解上一波为何顺利或危险。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `result-wave-screen` |
| 参考图 | `images/ui-reference-20260517/result-wave.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/result-wave-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 战斗波次胜利 |
| 退出方向 | 三选一奖励、阵图总览、主界面 |

## 玩家目标

- 查看本波击杀、突破、存活和收益。
- 理解防线表现。
- 进入造化奖励。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | “守阵回响”、波次成功提示。 |
| 中央 | 击杀、突破、存活、评级、获得资源。 |
| 左侧 | 花灵表现。 |
| 右侧 | 敌潮压力和建议。 |
| 底部 | 查看详情、返回主界面、进入造化。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `waveIndex` | int | 流程系统 | 波次显示。 |
| `killCount` | int | 战斗统计 | 击杀。 |
| `breachCount` | int | 战斗统计 | 突破。 |
| `aliveSpiritCount` | int | 敌人/阵容系统 | 存活花灵。 |
| `spiritCount` | int | 阵容系统 | 总花灵数量。 |
| `goldEarned` | int | 经济系统 | 本波灵砂。 |
| `expEarned` | int | 经济系统 | 本波识海。 |
| `grade` | string | 结算系统 | 波次评级。 |
| `adviceText` | string | 结算分析 | 建议。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 查看详情 | 点击 | 打开阵图总览，关闭后回结算。 |
| 进入造化 | 点击 | 进入三选一奖励界面。 |
| 返回主界面 | 点击 | 确认后退出本局回主界面。 |

## 无效输入

- 奖励已经生成或领取后，进入造化按钮不能重复触发。
- 非胜利结果不能显示进入造化。
- 结算数据缺失时显示 0 和明确占位，不报错。

## 迁移实现要点

- 结算数据应在战斗结束瞬间生成快照，避免后续清理敌人导致数据丢失。
- 波次结算和失败结算可以复用部分统计组件，但入口和按钮规则不同。
- 进入造化命令只由流程系统处理。

## 迁移验收

- 本波击杀、突破、存活和收益显示正确。
- 点击进入造化只执行一次。
- 从详情返回后仍停留在结算界面。
- 返回主界面后清理本局运行时对象。
