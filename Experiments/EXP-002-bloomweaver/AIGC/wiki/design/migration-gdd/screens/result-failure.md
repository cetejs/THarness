# 失败结算界面迁移策划案

## 定位

失败结算界面用于在防线崩溃后给出清晰反馈。它要说明失败原因，并给玩家重新开始或回主界面的出口。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `result-failure-screen` |
| 参考图 | `images/ui-reference-20260517/result-failure.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/result-failure-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 全部花灵倒下或防线失败 |
| 退出方向 | 开局界面、阵图总览、主界面 |

## 玩家目标

- 明确知道为什么输了。
- 查看本局统计。
- 选择重新开始或返回主界面。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | “灵圃失守”或失败标题。 |
| 中央 | 波次、击杀、突破、存活、失败原因。 |
| 中下 | 失败建议，例如补控制、提高治疗、缩短灵脉。 |
| 底部 | 查看详情、重新开始、返回主界面。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `waveIndex` | int | 流程系统 | 失败波次。 |
| `killCount` | int | 战斗统计 | 本局击杀。 |
| `breachCount` | int | 战斗统计 | 突破次数。 |
| `aliveSpiritCount` | int | 阵容系统 | 存活花灵。 |
| `failureReason` | string | 流程系统 | 失败原因。 |
| `adviceText` | string | 结算分析 | 改进建议。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 查看详情 | 点击 | 打开失败前阵图和战况快照。 |
| 重新开始 | 点击 | 清理旧局，回到开局或直接重开。 |
| 返回主界面 | 点击 | 清理旧局，回主界面。 |

## 无效输入

- 失败结算不能进入造化奖励。
- 重新开始执行中按钮锁定。
- 失败原因为空时显示默认文案“全部花灵倒下”。

## 迁移实现要点

- 失败快照必须在清理敌人前生成。
- 失败界面不应继续刷新实时战斗数据，避免数据跳变。
- 重开流程必须重置资源、奖励、商店、花灵等级和棋盘，除非未来设计局外保留项。

## 迁移验收

- 全部花灵倒下后进入失败结算。
- 失败界面不生成奖励。
- 重新开始后旧敌人、旧灵脉、旧奖励不残留。
- 返回主界面后可重新开始新局。
