# 商店界面迁移策划案

## 定位

商店界面是波间资源消费入口。玩家使用灵砂购买小幅强化，补足免费造化没有覆盖到的构筑方向。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `shop-screen` |
| 参考图 | `images/ui-reference-20260517/shop.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/shop-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 三选一奖励确认后 |
| 退出方向 | 下一波布阵、阵图总览 |

## 玩家目标

- 查看当前资源。
- 比较 6 个商品。
- 购买最多限定数量的商品。
- 刷新商品或进入下一波。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 商店标题、当前波次、灵砂、识海。 |
| 中央 | 2x3 商品卡。 |
| 右侧 | 商品详情、构筑推荐、购买预览。 |
| 底部 | 刷新商品、下一波观阵、总览入口。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `offers` | list | 商店系统 | 商品卡显示。 |
| `selectedOfferIndex` | int | UI 状态 | 当前选中商品。 |
| `spiritSand` | int | 经济系统 | 购买资源。 |
| `insightExp` | int | 经济系统 | 刷新资源。 |
| `purchasedCount` | int | 商店系统 | 当前商店购买次数。 |
| `maxPurchasesPerVisit` | int | 商店配置 | 购买上限。 |
| `refreshCost` | int | 商店配置 | 刷新消耗。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 商品卡 | 点击 | 高亮商品，刷新右侧预览。 |
| 购买按钮 | 点击 | 扣除灵砂，应用商品效果，商品标记已购。 |
| 刷新商品 | 点击 | 扣除刷新消耗，重新生成商品。 |
| 下一波观阵 | 点击 | 关闭商店，生成下一波棋盘并进入布阵。 |
| 阵图总览 | 点击 | 打开总览覆盖层，关闭后回商店。 |

## 无效输入

- 灵砂不足时购买按钮禁用。
- 商品已购买且不允许重复时按钮显示已购。
- 达到单次购买上限后其余购买按钮禁用。
- 刷新资源不足时刷新按钮禁用。

## 迁移实现要点

- 商品效果复用奖励类型，但商品配置独立保存价格。
- 商店价格应按波次缩放，当前基线为每波提高 15%。
- 购买效果同样写入运行时属性，不修改原始商品配置。
- 商店关闭后本次商品列表可以销毁，下次重新生成。

## 迁移验收

- 商品显示名称、类型、价格、效果。
- 资源足够时购买成功并扣除灵砂。
- 资源不足时不扣资源、不应用效果。
- 最多购买数量限制生效。
- 点击下一波后进入布阵，已购效果保留。
