# 设置界面迁移策划案

## 定位

设置界面用于调整音频、画面、操作和辅助选项。当前项目尚未完成设置持久化，因此迁移时应作为独立系统补齐。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `settings-screen` |
| 参考图 | `images/ui-reference-20260517/settings.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/settings-screen.prefab` |
| 目标技术 | 新引擎 UI 覆盖层或主页面 |
| 显示层级 | 覆盖层 |
| 来源阶段 | 主界面、暂停、其他可打开设置的界面 |
| 退出方向 | 来源界面 |

## 玩家目标

- 调整音量。
- 调整显示和操作偏好。
- 应用或取消修改。
- 恢复默认设置。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 左侧 | 分类页签：音频、画面、操作、战斗、辅助、其他。 |
| 右侧 | 当前分类的设置项。 |
| 底部 | 恢复默认、取消、应用、确认。 |

## 数据需求

| 字段 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| `sourceScreenKey` | string | UI 导航 | 关闭后返回。 |
| `selectedCategory` | enum | UI 状态 | 当前分类。 |
| `settingsDraft` | object | 设置系统 | 未应用修改。 |
| `settingsSaved` | object | 存档系统 | 已保存配置。 |
| `hasPendingChanges` | bool | 设置系统 | 应用按钮状态。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 分类按钮 | 点击 | 切换右侧设置项。 |
| 音量滑条 | 拖动 | 更新草稿值，可实时试听。 |
| 静音开关 | 点击 | 输出音量为 0，保留滑条值。 |
| 恢复默认 | 点击 | 当前分类或全部设置恢复默认。 |
| 取消 | 点击 | 丢弃未应用修改，回来源界面。 |
| 应用 | 点击 | 保存草稿，保持在设置界面。 |
| 确认 | 点击 | 保存草稿并关闭。 |

## 无效输入

- 没有未应用修改时应用按钮禁用。
- 设置保存失败时提示错误并保留草稿。
- 来源界面不存在时关闭后回主界面。

## 迁移实现要点

- 设置界面只能操作设置草稿，不直接写平台配置，应用时统一提交。
- 音频设置需要和目标引擎音频总线或混音器映射。
- 控制设置需要留出键位冲突检测。
- 设置需要持久化，但存档路径由目标引擎决定。

## 迁移验收

- 修改音量后应用按钮变可用。
- 应用后设置立即生效。
- 取消能回滚未应用修改。
- 重开游戏后已应用设置仍保留。
