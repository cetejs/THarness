# 开局界面迁移策划案

## 定位

开局界面负责让玩家确认本局出战配置。当前项目已经具备 5 名默认花灵和随机方案能力，迁移后应保留“选择本局守阵方案”的体验，不直接跳过到战斗。

## 基本信息

| 项 | 内容 |
| --- | --- |
| screen key | `opening-screen` |
| 参考图 | `images/ui-reference-20260517/opening.png` |
| 原 Unity Prefab | `Assets/Prefabs/UI/Screens/opening-screen.prefab` |
| 目标技术 | 新引擎 UI 系统 |
| 显示层级 | 主页面 |
| 来源阶段 | 主界面点击开始守阵 |
| 退出方向 | 主界面、布阵界面、上阵替换界面 |

## 玩家目标

- 查看并确认 5 名上阵花灵。
- 理解本局难度、挑战词条和初始构筑倾向。
- 使用随机方案快速开始。
- 进入布阵阶段。

## 视觉与布局

| 区域 | 内容 |
| --- | --- |
| 顶部 | 标题“灵圃启阵”和副标题。 |

| 左侧 | 难度选择和挑战词条。 |
| 中央 | 5 张花灵卡位。 |
| 右侧 | 构筑预览、初始阵图、初始造化。 |
| 底部 | 返回、随机方案、开始守阵。 |

## 交互规格

| 控件 | 操作 | 结果 |
| --- | --- | --- |
| 花灵卡位 | 点击 | 打开上阵替换界面，并传入槽位索引。 |
| 难度按钮 | 点击 | 设置难度，刷新敌潮压力和奖励预览。 |
| 挑战词条 | 点击 | 切换词条选中态，刷新构筑压力。 |
| 初始阵图 | 点击 | 设置初始阵图方案。 |
| 初始造化 | 点击 | 设置初始造化。 |
| `opening-random-button` | 点击 | 随机生成合法阵容和方案。 |
| `opening-back-button` | 点击 | 返回主界面。 |
| `opening-start-button` | 点击 | 初始化本局并进入布阵界面。 |

## 无效输入

- 阵容未满时不能开始守阵。
- 候选花灵不允许重复上阵时，重复选择要拒绝。
- 挑战词条超过上限时保持原状态并提示。
- 随机方案必须产出合法队伍，不能出现空槽。

## 迁移实现要点

- 开局界面只修改“本局开局配置”，不直接生成棋盘。
- 点击开始守阵后由流程系统统一创建棋盘、重置资源和进入布阵。
- 5 个花灵槽必须支持空态、已选态、禁用态和高亮态。
- 随机方案需要走同一套合法性校验，不能绕过规则。

## 迁移验收

- 点击开始守阵后进入布阵界面，且队伍正确传入战斗系统。
- 阵容未满时开始按钮不可用或提示原因。
- 替换花灵后开局界面立即刷新。
- 随机方案连续点击不会产生重复或空槽异常。
- 返回主界面不会残留半成品本局状态。
