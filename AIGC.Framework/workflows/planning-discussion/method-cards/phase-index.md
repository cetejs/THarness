# 策划阶段方法卡索引

| 策划阶段 | 优先方法卡 | AI 动作 |
| --- | --- | --- |
| 问清楚 | `cards/art-of-game-design.md` | 追问玩家幻想、核心动作、战斗中可感知结果。 |
| 拆体验 | `cards/designing-games.md` | 把题材、设定和口语表达转成体验结构。 |
| 建机制 | `cards/game-mechanics.md` | 拆资源、成长、循环、风险收益和平衡变量。 |
| 做反馈 | `cards/game-feel.md` | 补操作、命中、镜头、音效、特效和节奏反馈。 |
| 降低理解成本 | `cards/gamers-brain.md` | 检查引导、UI 表达、认知负担和玩家误读风险。 |
| 验证真实需求 | `cards/sprint.md`, `cards/mom-test.md` | 设计最小原型、测试问题和避免诱导式访谈。 |

## 调用规则

- 用户只给设定或题材时，从“问清楚”开始。
- 用户已经给出体验目标时，从“拆体验”开始。
- 用户已经给出机制雏形时，从“建机制”开始。
- 用户要求战斗、操作或爽感时，必须补充“做反馈”。
- 用户担心玩家看不懂、不会用或不愿复玩时，必须补充“降低理解成本”。
- 输出可开发方案前，必须补充“验证真实需求”。
