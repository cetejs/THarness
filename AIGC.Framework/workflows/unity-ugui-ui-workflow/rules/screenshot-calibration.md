# Unity UGUI UI 截图校准规则

截图校准用于把第一版估算施工结果迭代到可交付效果。

## 输入

- 目标 UI 效果图。
- Unity 实际运行截图。
- 当前 UGUI 层级。
- 当前 RectTransform 布局表。
- UI 资源整理员拼接还原文档。
- 用户确认的还原目标。

## 校准前提判断

开始截图校准前，必须判断：

1. 目标图和 Unity 截图是否为同一界面。
2. Unity 截图是否包含完整 Canvas。
3. 当前 UGUI 层级是否与施工文档一致。
4. 是否提供当前 RectTransform 布局表。
5. UI 资源整理员资源是否为已审核资源。

如果缺少当前布局表，只能输出视觉偏差建议，不能输出确定的新 RectTransform。

## 必须输出

- 位置偏差表。
- 尺寸偏差表。
- 透明度修正表。
- 层级修正表。
- 字体字号颜色修正表。
- 新的 RectTransform 参数。
- 下一轮优先修改清单。
- 回派角色。

## 修改批次

截图校准必须按批次输出，不要一次性混改所有内容：

| 批次 | 修改内容 |
| --- | --- |
| 第一批 | 背景、主构图、Logo、主按钮组 |
| 第二批 | 图标、装饰、版本号、辅助文字 |
| 第三批 | 字体、字号、颜色、透明度 |
| 第四批 | Hover、Pressed、Disabled、打开关闭动画 |

每一批都必须说明是否需要用户再次截图确认。

## 回派规则

| 问题类型 | 回派角色 |
| --- | --- |
| 已切元素缺失、边缘被裁、背景残留、透明通道错误 | UI 资源整理员 |
| 资源命名不清、Sprite 状态缺失 | UI 资源整理员 |
| 节点层级错误、RectTransform 错误 | UI 开发者 |
| Button 事件未注册、字段未绑定 | UI 开发者 |
| 功能语义不清、点击后去哪里不明确 | UI 开发者先确认用户决策 |
| 适配目标变化 | UI 开发者先确认，再派发 |

## 校准要求

- 第一版通常只要求达到用户可判断方向的还原度，不把估算结果当最终结果。
- 所有修正参数必须能追溯到目标图和 Unity 截图的对比。
- 如果目标图本身缺少素材或字体信息，必须继续保留待确认项。
- 不通过截图校准改 UI 功能语义。

## 成功标准

1. 每个明显偏差都有节点名、问题、当前值和建议修正。
2. 每个修正项都有明确回派角色。
3. 下一轮修改范围可执行。
4. 未确认项没有被静默解决。
