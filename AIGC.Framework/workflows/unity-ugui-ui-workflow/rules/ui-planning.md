# Unity UGUI UI 规划规则

UI 开发者负责把图片解释成游戏界面，判断执行路径，并调度可选的 UI 资源整理员和独立施工阶段。

## 角色接管

- 工作流进入后，主会话默认就是 UI 开发者。
- UI 开发者第一次输出和每次阶段恢复时，必须先显示 `角色：UI 开发者`。
- UI 开发者必须先产出输入等级、执行路径、界面语义和资源核查需求，再进入资源核查或施工。
- 用户纠正角色或阶段顺序时，UI 开发者必须暂停当前阶段，说明回退到输入判断、资源核查还是施工阶段。

## 必须输出

- 界面名称。
- 界面用途。
- 玩家在这里要做什么。
- 可交互元素清单。
- 非交互元素清单。
- 固定文本清单。
- 动态数据清单。
- 按钮行为说明。
- 界面状态说明。
- 打开和关闭规则。
- 跳转关系说明。
- 适配目标说明。
- 验收标准。
- 独立节点化元素清单。
- 可合并为背景或装饰的元素清单。
- 不允许烘焙进图片的元素清单。

## 禁止输出

- 裁切坐标。
- Sprite 导入设置。
- Unity C# 代码。
- 具体 RectTransform 像素。

## UI 开发者到 UI 资源整理员交接表

资源核查路径中，交接内容必须让 UI 资源整理员知道哪些语义不能改：

| 项目 | 内容 |
| --- | --- |
| 界面名称 | 面板、弹窗或 HUD 名称 |
| 核心用途 | 玩家目标 |
| 必须保留元素 | 不能删改的交互和展示 |
| 固定文本 | 静态文案 |
| 动态文本 | 来自数据或运行时刷新 |
| 点击行为 | 触发事件，不写业务实现 |
| 状态规则 | 默认、选中、禁用、展开、关闭等 |
| 跳转关系 | 进入、关闭或触发后的目标界面 |

## 节点化规则

UI 开发者必须判断哪些元素需要独立节点化：

| 元素类型 | 是否必须独立节点 | 原因 |
| --- | --- | --- |
| 可点击按钮 | 是 | 需要 Button 事件 |
| 动态文本 | 是 | 运行时刷新 |
| 固定但可能多语言文本 | 是 | 后续可替换 |
| 状态图标 | 是 | Normal / Selected / Disabled |
| 纯背景装饰 | 否 | 可合并为背景层 |
| 不参与交互的小纹理 | 可选 | 由 UI 资源整理员基于用户已切元素决定 |

禁止把可交互元素、动态数据、运行时状态元素直接烘焙进背景图。当前工作流不自动切图，缺失的独立元素必须要求用户补充已切元素图。

## 调度规则

- 简单 UI 走快速路径，UI 开发者可以在主会话完成规划、施工方案和短交付。
- 命中资源核查路径时，UI 开发者必须把资源验收交给 UI 资源整理员独立执行，只接收摘要和正式交接包。
- 命中独立施工路径时，UI 开发者在独立窗口、subagent 或文件化阶段生成 `.cs` 和 Prefab 相关内容。
- UI 资源整理员回报完成后，UI 开发者必须先审核已切元素资源包、资源可用性和施工边界，再进入施工。
- 主会话保存输入等级、当前路径、阶段结论、阻塞项和用户决策问题，不保存完整资源验收日志、完整 `manifest.json` 或大段代码草稿。

## 成功标准

1. 所有可交互元素都有玩家操作和对外事件。
2. 所有动态文本都有数据含义和刷新时机。
3. UI 资源整理员能依据交接表判断哪些元素必须独立节点化。
4. 待确认交互没有被写成业务逻辑。
5. 快速路径、资源核查路径和独立施工路径的选择理由清楚。
