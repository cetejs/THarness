# Unity UGUI UI 上下文隔离规则

复杂 UI 不应把资源清单、manifest、代码和截图校准细节全部塞进主会话。主会话只保留决策摘要，长内容写入文件或交给独立执行单元。

## 触发条件

| 触发条件 | 推荐处理 |
| --- | --- |
| 已切元素数量 > 20 | 激活 `ui-asset-curator`。 |
| 图片总数 > 10 或 manifest 很长 | 激活 `ui-asset-curator`，只回传摘要。 |
| 需要生成 `.cs` 和 Prefab | `ui-programmer` 在独立窗口、subagent 或文件化阶段执行，不在 PL 主会话展开代码。 |
| 截图校准超过 1 轮 | 每轮输出文件化偏差表，主会话只读摘要。 |
| 交付报告超过 200 行 | 写入文档文件，回复中只给摘要。 |
| 资源清单很长 | 资源表写文件，不内联到主会话。 |
| 多页签、多弹窗或大量按钮状态 | 按页签、弹窗或状态组拆分文档，主会话只保留当前范围。 |
| 用户连续修正 UI 语义 | 每轮修正写入变更记录，主会话只保留最新决策。 |

## 主会话保留内容

主会话只保留：

- 输入等级。
- 当前路径：快速路径 / 资源核查路径 / 独立施工路径。
- 阶段结论。
- 阻塞项。
- 用户需要决策的问题。
- 产物路径和验证结果。

主会话不保留：

- 全量资源表。
- 全量 manifest。
- 大段 Editor 或 Controller 代码。
- 多轮截图的完整偏差明细。
- 已被新版语义覆盖的旧讨论过程。

## 文件化产物

长内容必须写入独立产物：

| 内容 | 推荐产物 |
| --- | --- |
| 资源验收明细 | 资源验收文档 |
| 拼装位置和节点映射 | 拼装还原文档 |
| Prefab、脚本和绑定说明 | Prefab 施工报告 |
| 每轮截图对比 | 截图校准表 |
| Controller / Editor 代码 | `.cs` 脚本文件 |
| 用户语义修正 | UI 语义变更记录 |

具体写入位置必须来自目标项目适配层、项目 wiki 或用户确认；通用 AIGC 不能保存具体界面事实、资源路径或运行记录。

## 独立执行规则

- `ui-asset-curator` 只接收资源核查所需输入：UI 规划摘要、已切元素目录、目标资源命名规则、输出目录和待确认项。
- `ui-asset-curator` 只回传摘要：`Final` 数量、`ReferenceOnly` 数量、缺失/不可用资源、manifest 路径、拼装文档路径和阻塞项。
- `ui-programmer` 生成 `.cs` 和 Prefab 相关内容时，优先写入脚本和施工报告，不把完整代码贴回主会话。
- 多轮截图校准只在主会话报告本轮结论、产物路径、阻塞项和下一轮动作。

## 成功标准

1. 主会话能在不读取长明细的情况下判断当前状态。
2. 所有长表、长代码和多轮校准细节都有文件化产物。
3. 独立执行单元的输入和输出都是短交接包。
4. 项目事实没有写入通用 AIGC。
