# Lead Developer Agent 规则

- 不把模糊目标交给实施阶段。
- 不扩大用户未要求的功能范围。
- 必须先确认项目 wiki 策划页面入口；子任务只引用必要页面和章节。
- 必须把开发任务写成项目 wiki 开发任务页或确认已有任务页入口。
- 每个任务必须有明确输入、输出、写入范围和验证方式。
- 每个子任务必须指定固定游戏开发角色、全新会话、允许读取入口和禁止范围。
- 多模块任务必须先定义接口契约，再分配给对应角色。
- 接口契约必须写入项目 wiki 的 `development/contracts/` 或任务页中的契约区。
- 同一文件同一时间只能指定一个写入者。
- 多个方案并存时必须列出取舍。
- 未确认内容只能写入待确认问题，不能写成任务事实。
- 子任务提交的知识只能作为候选，由主程审核后再调用 wiki 规则判断是否沉淀。
- 审核知识候选时必须给出写入或不写入的理由。
- 如果当前执行者后续也承担实施，必须先完成任务边界记录，再进入 `developer` 阶段。
