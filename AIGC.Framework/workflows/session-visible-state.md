# 会话工作流标识规则

## 用途

让用户在会话中明确知道当前 AI 正按哪个工作流执行。

## 显示格式

只显示一行：

```text
【当前工作流】{workflow_id} / {中文名称}
```

当前已知工作流：

| workflow_id | 中文名称 |
| --- | --- |
| `planning-discussion` | 策划案讨论 |
| `development` | 开发执行 |
| `project-wiki-maintenance` | 项目 Wiki 维护 |
| `capability-evolution` | AIGC 能力演化 |

## 显示时机

- 工作流刚触发时。
- 工作流发生切换时。
- 长任务恢复、上下文恢复或阶段性继续时。

## 禁止

- 不默认展示阶段。
- 不默认展示本轮目标。
- 不默认展示成功标准。
- 不默认展示写入边界。
- 不把内部执行细节暴露成会话状态条。
