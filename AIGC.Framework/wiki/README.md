# 通用架构 Wiki

本目录是 OneHarness 的通用架构知识库，只服务通用项目架构检索。

它借鉴外部 harness 的轻量信息体系：每个知识页用 YAML 头声明 `id`、`title`、`summary`、`tags`、`relates`、`status` 和 `read_when`，入口索引只负责路由，不复制正文。

## 目标

- 用最少 token 找到架构搭建所需的通用知识。
- 把项目结构、模块边界、依赖方向、配置入口、数据流等通用架构问题拆成可检索卡片。
- 保持通用框架和具体项目资料隔离。

## 非目标

- 不保存项目 wiki。
- 不保存项目运行记录。
- 不保存项目决策事实。
- 不保存项目代码结构分析。
- 不保存某个引擎、业务、玩法或产品的专用资料。

## 工作方式

1. 先读 `INDEX.md`。
2. 按 `read_when` 命中一个规则或架构分区。
3. 只读取命中的知识页。
4. 产出或修改后执行最小健康检查。

## 设计取舍

- 保留轻量 YAML 头和关联索引思想。
- 废弃旧 `raw/knowledge/log/html` 混合式项目 wiki。
- 通用知识只写稳定架构模式；项目事实进入目标项目适配层。
